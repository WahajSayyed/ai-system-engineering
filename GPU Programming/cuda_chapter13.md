# Chapter 13 — Custom CUDA Extensions for PyTorch

## Learning objectives

By the end of this chapter you will:

- Understand the PyTorch extension mechanism and how it bridges Python ↔ CUDA C
- Write a complete CUDA C kernel with a C++ host wrapper
- Bind it to Python using `torch.utils.cpp_extension`

- Benchmark your extension against PyTorch built-ins
- Debug common extension build and runtime errors

## 13.1 Why write PyTorch extensions?

PyTorch covers the vast majority of operations you need. You write a
custom extension when:

```
Reason                          Example
──────────────────────────────────────────────────────────────────────
Kernel fusion                   Fused LayerNorm (mean+var+norm in 1 pass)
Non-standard activation         Mish, SiLU variant with custom backward
Custom attention variant        Sliding window, linear, sparse attention
Quantisation ops                INT8 GEMM with custom dequantisation
Research novelty                New layer type not in PyTorch yet
Maximum performance             Hand-tuned kernel for your exact shapes
```

The output of this chapter is the exact workflow used by teams building
Flash Attention, xFormers, bitsandbytes, and Triton.

## 13.2 The extension architecture

```
Python (your training code)
    │
    │  import my_extension
    ▼
Python binding layer  (pybind11, auto-generated by PyTorch)
    │
    │  my_extension.forward(x, weight)
    ▼
C++ host function     (dispatches to CUDA kernel, handles tensors)
    │
    │  my_kernel<<<grid, block>>>(x.data_ptr<float>(), ...)
    ▼
CUDA kernel           (runs on GPU, written in CUDA C)
    │
    ▼
GPU hardware
```

Three files for a typical extension:

```
my_extension/
├── my_kernel.cu        ← CUDA C kernel + C++ dispatch function
├── my_extension.cpp    ← pybind11 module definition
└── setup.py            ← build configuration
```

## 13.3 A complete worked example — fused ReLU + scale

We will implement `out = ReLU(x) * scale` as a single fused kernel,
with a correct backward pass. This is simple enough to understand fully
but real enough to demonstrate every component.

### Step 1 — The CUDA kernel (`relu_scale.cu`)

```cpp
// relu_scale.cu
#include <torch/extension.h>
#include <cuda.h>
#include <cuda_runtime.h>

// ── Forward kernel ────────────────────────────────────────────────────────────
__global__ void relu_scale_forward_kernel(
        const float* __restrict__ x,
        float* __restrict__ out,
        float scale,
        int n) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i < n) {
        float val = x[i];
        out[i]    = (val > 0.0f) ? val * scale : 0.0f;
    }
}

// ── Backward kernel ───────────────────────────────────────────────────────────
// d_x     = grad_out * scale  if x > 0, else 0
// d_scale = sum(grad_out * relu(x))   (scalar gradient)
__global__ void relu_scale_backward_kernel(
        const float* __restrict__ grad_out,
        const float* __restrict__ x,
        float* __restrict__ grad_x,
        float* __restrict__ grad_scale_partial,   // one value per block
        float scale,
        int n) {
    extern __shared__ float smem[];

    int i   = blockIdx.x * blockDim.x + threadIdx.x;
    int tid = threadIdx.x;

    float local_grad_scale = 0.0f;

    if (i < n) {
        float xi  = x[i];
        float go  = grad_out[i];
        float mask = (xi > 0.0f) ? 1.0f : 0.0f;

        // Gradient w.r.t. x
        grad_x[i] = go * scale * mask;

        // Partial gradient w.r.t. scale (accumulate in shared memory)
        local_grad_scale = go * xi * mask;
    }

    // Reduce grad_scale partial sums within block
    smem[tid] = local_grad_scale;
    __syncthreads();

    for (int stride = blockDim.x / 2; stride > 0; stride >>= 1) {
        if (tid < stride)
            smem[tid] += smem[tid + stride];
        __syncthreads();
    }

    if (tid == 0)
        grad_scale_partial[blockIdx.x] = smem[0];
}

// ── C++ host functions (called from Python via pybind11) ──────────────────────
torch::Tensor relu_scale_forward(torch::Tensor x, float scale) {
    // Input validation
    TORCH_CHECK(x.is_cuda(),               "x must be a CUDA tensor");
    TORCH_CHECK(x.is_contiguous(),         "x must be contiguous");
    TORCH_CHECK(x.scalar_type() == torch::kFloat32, "x must be float32");

    int n   = x.numel();
    auto out = torch::empty_like(x);

    int threads = 256;
    int blocks  = (n + threads - 1) / threads;

    relu_scale_forward_kernel<<<blocks, threads>>>(
        x.data_ptr<float>(),
        out.data_ptr<float>(),
        scale,
        n
    );

    // Check for CUDA errors (debug builds)
    C10_CUDA_KERNEL_LAUNCH_CHECK();

    return out;
}

std::vector<torch::Tensor> relu_scale_backward(
        torch::Tensor grad_out,
        torch::Tensor x,
        float scale) {
    TORCH_CHECK(grad_out.is_cuda() && x.is_cuda(), "inputs must be CUDA tensors");
    TORCH_CHECK(grad_out.is_contiguous() && x.is_contiguous(), "inputs must be contiguous");

    int n       = x.numel();
    auto grad_x = torch::empty_like(x);

    int threads = 256;
    int blocks  = (n + threads - 1) / threads;
    int smem    = threads * sizeof(float);

    // Partial sums for grad_scale (one per block, reduced on CPU)
    auto grad_scale_partial = torch::zeros({blocks}, x.options());

    relu_scale_backward_kernel<<<blocks, threads, smem>>>(
        grad_out.data_ptr<float>(),
        x.data_ptr<float>(),
        grad_x.data_ptr<float>(),
        grad_scale_partial.data_ptr<float>(),
        scale,
        n
    );

    C10_CUDA_KERNEL_LAUNCH_CHECK();

    // Reduce partial sums to get scalar grad_scale
    // (using PyTorch's built-in sum — avoids another custom kernel)
    torch::Tensor grad_scale = grad_scale_partial.sum();

    return {grad_x, grad_scale};
}
```

### Step 2 — pybind11 bindings (`relu_scale_bind.cpp`)

```cpp
// relu_scale_bind.cpp
#include <torch/extension.h>

// Forward declarations (defined in relu_scale.cu)
torch::Tensor relu_scale_forward(torch::Tensor x, float scale);
std::vector<torch::Tensor> relu_scale_backward(
    torch::Tensor grad_out, torch::Tensor x, float scale);

// PYBIND11_MODULE creates the Python module
PYBIND11_MODULE(TORCH_EXTENSION_NAME, m) {
    m.def("forward",  &relu_scale_forward,  "ReLU+Scale forward  (CUDA)");
    m.def("backward", &relu_scale_backward, "ReLU+Scale backward (CUDA)");
}
```

## 13.4 Autograd integration — the `Function` class

Raw extension functions don't participate in PyTorch's autograd graph.
Wrap them in `torch.autograd.Function` to make them differentiable:

```python
# relu_scale_op.py
import torch
import relu_scale_cuda   # your compiled extension

class ReLUScaleFunction(torch.autograd.Function):

    @staticmethod
    def forward(ctx, x, scale):
        """
        ctx: context object — store tensors needed for backward
        x:   input tensor (CUDA float32)
        scale: Python float (not a tensor here — gradient handled separately)
        """
        # Validate inputs
        assert x.is_cuda, "Input must be on GPU"
        assert x.dtype == torch.float32, "Input must be float32"

        # Save for backward pass
        ctx.save_for_backward(x)
        ctx.scale = scale

        # Call our CUDA kernel
        out = relu_scale_cuda.forward(x, scale)
        return out

    @staticmethod
    def backward(ctx, grad_output):
        """
        grad_output: upstream gradient (same shape as forward output)
        Returns: gradients for each forward() input (x, scale)
        """
        x,     = ctx.saved_tensors
        scale  = ctx.scale

        # Ensure grad_output is contiguous (upstream ops may have strided it)
        grad_output = grad_output.contiguous()

        # Call our backward CUDA kernel
        grad_x, grad_scale = relu_scale_cuda.backward(grad_output, x, scale)

        # Return one gradient per forward() argument
        # None for non-tensor arguments that don't need gradients
        return grad_x, grad_scale


# Convenience wrapper — looks like a regular PyTorch function
def relu_scale(x: torch.Tensor, scale: float) -> torch.Tensor:
    return ReLUScaleFunction.apply(x, scale)


# ── Test autograd ──────────────────────────────────────────────────────────────
if __name__ == '__main__':
    x     = torch.randn(1000, requires_grad=True, device='cuda')
    scale = torch.tensor(2.0, requires_grad=True, device='cuda')

    # Forward
    out  = relu_scale(x, float(scale))
    loss = out.sum()

    # Backward
    loss.backward()

    print(f"x.grad shape:     {x.grad.shape}")     # (1000,)
    print(f"scale.grad:       {scale.grad}")        # scalar

    # Gradient check — compare numerical vs analytical gradient
    from torch.autograd import gradcheck
    x_double = torch.randn(100, dtype=torch.float64,
                            requires_grad=True, device='cuda')
    # gradcheck needs float64 for numerical precision
    result = gradcheck(
        lambda inp: relu_scale(inp.float(), 2.0).double(),
        (x_double,),
        eps=1e-4, atol=1e-3, rtol=1e-3
    )
    print(f"gradcheck passed: {result}")
```

## 13.5 JIT compilation — no `setup.py` needed

For iteration during development, use `torch.utils.cpp_extension.load`
to compile inline without a `setup.py`:

```python
import torch
from torch.utils.cpp_extension import load

# JIT compile — runs on first import, cached thereafter (~30 seconds first time)
relu_scale_jit = load(
    name='relu_scale_jit',
    sources=['relu_scale.cu', 'relu_scale_bind.cpp'],
    extra_cuda_cflags=['-O3', '--use_fast_math', '-arch=sm_75'],
    verbose=True   # print compilation output
)

# Now use it identically to the setup.py-built version
x   = torch.randn(1_000_000, device='cuda')
out = relu_scale_jit.forward(x, 2.0)
print(out[:5])
```

JIT compilation is ideal for:
- Development iterations (no reinstall needed)
- Notebooks and Colab
- Extensions with a single file

`setup.py` is better for:
- Production deployment (compile once, import fast)
- Extensions with multiple files or complex dependencies
- Packages distributed via pip

## 13.6 Inline CUDA strings — fastest prototyping

For the fastest prototyping loop (no external files at all), use
`torch.utils.cpp_extension.load_inline`:

```python
import torch
from torch.utils.cpp_extension import load_inline

cuda_src = r"""
__global__ void swish_kernel(const float* x, float* out, int n) {
    // Swish: x * sigmoid(x) = x / (1 + exp(-x))
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i < n) {
        float val = x[i];
        out[i] = val / (1.0f + expf(-val));
    }
}

torch::Tensor swish_forward(torch::Tensor x) {
    TORCH_CHECK(x.is_cuda() && x.is_contiguous(), "bad input");
    int n   = x.numel();
    auto out = torch::empty_like(x);
    int threads = 256, blocks = (n + 255) / 256;
    swish_kernel<<<blocks, threads>>>(
        x.data_ptr<float>(), out.data_ptr<float>(), n);
    C10_CUDA_KERNEL_LAUNCH_CHECK();
    return out;
}
"""

cpp_src = r"""
torch::Tensor swish_forward(torch::Tensor x);
PYBIND11_MODULE(TORCH_EXTENSION_NAME, m) {
    m.def("forward", &swish_forward, "Swish forward (CUDA)");
}
"""

swish_ext = load_inline(
    name='swish_inline',
    cuda_sources=[cuda_src],
    cpp_sources=[cpp_src],
    extra_cuda_cflags=['-O3', '--use_fast_math'],
    verbose=False
)

# Use it
x   = torch.randn(10_000_000, device='cuda')
out = swish_ext.forward(x)

# Verify against PyTorch reference
ref = x * torch.sigmoid(x)
print(f"Max error: {(out - ref).abs().max():.2e}")
```

## 13.7 Handling multiple dtypes with AT_DISPATCH

Real extensions need to handle float16, bfloat16, and float32.
`AT_DISPATCH_FLOATING_TYPES_AND_HALF` generates specialised code for each:

```cpp
// In your .cu file — type-generic kernel
template <typename scalar_t>
__global__ void relu_scale_generic_kernel(
        const scalar_t* __restrict__ x,
        scalar_t* __restrict__ out,
        float scale,
        int n) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i < n) {
        scalar_t val = x[i];
        out[i] = (val > scalar_t(0)) ? scalar_t(float(val) * scale)
                                      : scalar_t(0);
    }
}

torch::Tensor relu_scale_forward_generic(torch::Tensor x, float scale) {
    TORCH_CHECK(x.is_cuda() && x.is_contiguous(), "bad input");

    auto out    = torch::empty_like(x);
    int n       = x.numel();
    int threads = 256;
    int blocks  = (n + threads - 1) / threads;

    // Dispatch to correct dtype specialisation at runtime
    AT_DISPATCH_FLOATING_TYPES_AND_HALF(x.scalar_type(), "relu_scale_forward", [&] {
        relu_scale_generic_kernel<scalar_t><<<blocks, threads>>>(
            x.data_ptr<scalar_t>(),
            out.data_ptr<scalar_t>(),
            scale,
            n
        );
    });

    C10_CUDA_KERNEL_LAUNCH_CHECK();
    return out;
}
```

## 13.8 Benchmarking your extension

```python
import torch
import time
import numpy as np

def benchmark_cuda(fn, *args, n_warmup=10, n_rep=100):
    """Benchmark a CUDA operation using CUDA events."""
    for _ in range(n_warmup):
        fn(*args)
    torch.cuda.synchronize()

    times = []
    for _ in range(n_rep):
        start = torch.cuda.Event(enable_timing=True)
        end   = torch.cuda.Event(enable_timing=True)
        start.record()
        fn(*args)
        end.record()
        torch.cuda.synchronize()
        times.append(start.elapsed_time(end))
    return np.array(times)


N = 50_000_000
x = torch.randn(N, device='cuda')

# Your custom kernel
times_custom = benchmark_cuda(lambda: relu_scale(x, 2.0))

# PyTorch reference (two separate ops)
times_pytorch = benchmark_cuda(lambda: torch.relu(x) * 2.0)

# PyTorch fused (if available)
times_fused = benchmark_cuda(lambda: torch.nn.functional.relu(x).mul_(2.0))

print(f"{'Method':<25} {'Mean(ms)':>10} {'Min(ms)':>10} {'GB/s':>8}")
print("-" * 58)
for name, t in [("Custom kernel",    times_custom),
                ("PyTorch 2 ops",    times_pytorch),
                ("PyTorch in-place", times_fused)]:
    mean_t = t.mean()
    min_t  = t.min()
    # 2 reads (x read twice in naive) + 1 write = 3N floats
    # Custom: 1 read + 1 write = 2N floats
    bw = 2 * N * 4 / (min_t * 1e-3) / 1e9
    print(f"{name:<25} {mean_t:>10.3f} {min_t:>10.3f} {bw:>8.1f}")
```

```

──────────────────────────────────────────────────────────────────────────

image is available for execution       in setup.py              (sm_75 for T4, sm_80 for A100)

                                       .cpp binding file        or add to PYBIND11_MODULE

RuntimeError: Sizes of tensors must    Shape mismatch in        Add TORCH_CHECK for shapes
```

## 13.10 A more complex example — fused LayerNorm forward

LayerNorm is the canonical fusion target: mean, variance, normalise, scale,
and shift are five operations that touch the same data. Fused into one kernel,
we save four HBM passes.

```cpp
// layer_norm_fwd.cu  (simplified — 1D over last dimension)
#include <torch/extension.h>
#include <cuda.h>

__global__ void layer_norm_forward_kernel(
        const float* __restrict__ x,      // (B, N)
        const float* __restrict__ gamma,  // (N,)
        const float* __restrict__ beta,   // (N,)
        float* __restrict__ out,          // (B, N)
        float* __restrict__ mean_out,     // (B,)
        float* __restrict__ rstd_out,     // (B,) reciprocal std
        int N, float eps) {
    /*
     * One block per row (batch element).
     * Uses warp shuffles for fast mean and variance reduction.
     */
    extern __shared__ float smem[];

    int row = blockIdx.x;
    int tid = threadIdx.x;

    const float* row_x   = x   + row * N;
    float*       row_out = out + row * N;

    // ── Pass 1: compute mean ──────────────────────────────────────────────
    float sum = 0.0f;
    for (int i = tid; i < N; i += blockDim.x)
        sum += row_x[i];

    // Warp reduce
    for (int d = 16; d > 0; d >>= 1)
        sum += __shfl_down_sync(0xffffffff, sum, d);
    if ((tid % 32) == 0) smem[tid / 32] = sum;
    __syncthreads();

    if (tid < (blockDim.x / 32)) sum = smem[tid];
    for (int d = 16; d > 0; d >>= 1)
        sum += __shfl_down_sync(0xffffffff, sum, d);

    float mean = sum / N;
    if (tid == 0) mean_out[row] = mean;

    // ── Pass 2: compute variance ──────────────────────────────────────────
    float var = 0.0f;
    for (int i = tid; i < N; i += blockDim.x) {
        float diff = row_x[i] - mean;
        var += diff * diff;
    }

    for (int d = 16; d > 0; d >>= 1)
        var += __shfl_down_sync(0xffffffff, var, d);
    if ((tid % 32) == 0) smem[tid / 32] = var;
    __syncthreads();

    if (tid < (blockDim.x / 32)) var = smem[tid];
    for (int d = 16; d > 0; d >>= 1)
        var += __shfl_down_sync(0xffffffff, var, d);

    float rstd = rsqrtf(var / N + eps);
    if (tid == 0) rstd_out[row] = rstd;

    // ── Pass 3: normalise + scale + shift ─────────────────────────────────
    for (int i = tid; i < N; i += blockDim.x) {
        float norm    = (row_x[i] - mean) * rstd;
        row_out[i]    = gamma[i] * norm + beta[i];
    }
}

torch::Tensor layer_norm_forward(
        torch::Tensor x,
        torch::Tensor gamma,
        torch::Tensor beta,
        float eps = 1e-5) {
    TORCH_CHECK(x.is_cuda() && x.is_contiguous(), "x must be CUDA contiguous");
    int B = x.size(0);
    int N = x.size(1);

    auto out      = torch::empty_like(x);
    auto mean_out = torch::empty({B}, x.options());
    auto rstd_out = torch::empty({B}, x.options());

    int threads   = std::min(N, 1024);
    int smem      = (threads / 32) * sizeof(float);

    layer_norm_forward_kernel<<<B, threads, smem>>>(
        x.data_ptr<float>(),
        gamma.data_ptr<float>(),
        beta.data_ptr<float>(),
        out.data_ptr<float>(),
        mean_out.data_ptr<float>(),
        rstd_out.data_ptr<float>(),
        N, eps
    );

    C10_CUDA_KERNEL_LAUNCH_CHECK();
    return out;
}
```

This kernel does 3 passes over the row data (mean, var, normalise) vs
5 separate PyTorch operations. The bandwidth saving is significant for
large hidden dimensions (N=4096, 8192 typical in LLMs).

## Chapter summary

| Concept | Key takeaway |
|---|---|
| Extension anatomy | `.cu` kernel + `.cpp` binding + `setup.py` — three files, one import |
| `TORCH_CHECK` | Always validate: `.is_cuda()`, `.is_contiguous()`, `.scalar_type()` |
| `.data_ptr<float>()` | Raw pointer to tensor data — only valid while tensor is alive |
| `ctx.save_for_backward` | Tensors needed in backward — don't store raw pointers |
| `AT_DISPATCH` | Generates dtype specialisations — required for FP16/BF16 support |
| `load_inline` | Fastest iteration: compile from string, no files needed |
| `gradcheck` | Numerical gradient verification — always run before trusting backward |
| `.contiguous()` | Call before `.data_ptr()` — non-contiguous tensors give wrong results |
| `C10_CUDA_KERNEL_LAUNCH_CHECK()` | Catches async CUDA errors immediately — put after every launch |
| Fusion benefit | LayerNorm: 5 passes → 3 passes → fewer HBM round-trips → faster |
