# AI System Engineering

A personal, self-directed learning hub for AI system engineering and software development — spanning ML/DL fundamentals, GPU programming, LLM serving/orchestration, and applied tooling.

This repo is a working knowledge base rather than a polished course. Most material here is generated with the help of Claude (Anthropic) as a study aid — structured into chapters, guides, or cookbooks — then used, tested, and refined hands-on. Expect it to keep growing as new topics get added.

## What's inside

| Folder | Topic |
|---|---|
| [`Machine Learnig Chapters`](./Machine%20Learnig%20Chapters) | Classical ML through deep learning fundamentals, chapter by chapter |
| [`PyTorch`](./PyTorch) | PyTorch from foundations to advanced topics (fine-tuning, LoRA, GRPO, multi-node training) |
| [`GPU Programming`](./GPU%20Programming) | CUDA, Triton, PTX/SASS, CUDA Graphs, and low-level GPU performance work |
| [`FastAPI-FastBook`](./FastAPI-FastBook) | FastAPI course — building production-style APIs, async patterns, background jobs |
| [`LangGraph-cookbook`](./LangGraph-cookbook) | Recipes for building agents and workflows with LangGraph |
| [`vLLM-cookbook`](./vLLM-cookbook) | Serving and inference patterns with vLLM |
| [`OpenCV`](./OpenCV) | Computer vision fundamentals and applied CV (shape/object detection, pipelines) |
| [`ClaudeCodeGuide`](./ClaudeCodeGuide) | Notes and guides on working with Claude Code |

Each folder is largely self-contained, with material organized as sequential chapters or standalone guides/notebooks depending on the topic.

## Why this exists

A single place to:
- Learn new AI/ML and systems topics in a structured, code-first way
- Keep reference material tied to real projects (robotics automation, GPU infra, LLM fine-tuning pipelines) rather than abstract theory
- Revisit and reuse notes instead of re-deriving them each time

## How the material is built

Most content is drafted with Claude as a starting point — outlines, explanations, and code examples — then reviewed and adapted while working through the topic. It's meant for learning and reference, not as an authoritative or fully audited source. If something looks off, treat it as a prompt to dig deeper rather than ground truth.

## Status

🚧 Actively growing. New folders and chapters get added as new topics come up — there's no fixed roadmap or completion date.
